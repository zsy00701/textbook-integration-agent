# API 接口文档

> 所有接口均在 `/api/*` 前缀下;Content-Type 默认 `application/json`(上传除外)。

## 1. 健康检查

### `GET /api/health`
**响应**
```json
{
  "status": "ok",
  "model": "deepseek-chat",
  "embedding": "BAAI/bge-small-zh-v1.5",
  "textbooks": 7,
  "parsed": 7,
  "graphs": 7
}
```

---

## 2. 教材管理

### `POST /api/upload`
**请求**:`multipart/form-data`,`files` 字段可重复(批量上传)
**响应**
```json
{
  "accepted": [{"book_id": "03_生理学", "filename": "03_生理学.pdf"}],
  "skipped": [{"filename": "x.zip", "reason": "不支持的格式 .zip"}]
}
```

### `GET /api/textbooks`
**响应**:`TextbookMeta[]`
```json
[
  {
    "textbook_id": "03_生理学",
    "filename": "03_生理学.pdf",
    "title": "03_生理学",
    "total_pages": 450,
    "total_chars": 638000,
    "chapter_count": 22,
    "status": "ready",
    "error": null
  }
]
```
status 取值:`parsing | parsed | extracting | ready | failed`

### `POST /api/scan`
扫描 `data/textbooks/` 下未解析文件,触发解析(开发/赛方便利接口)。
**响应**:`{"scanned": 7}`

### `DELETE /api/textbooks/{book_id}`
**响应**:`{"removed": ["03_生理学.json", "..."]}`

## 3. 知识图谱

### `POST /api/graph/{book_id}/extract`
对单本教材触发知识点抽取。
**响应**:`{"status": "started", "book_id": "..."}`(异步)

### `POST /api/graph/extract_all`
对所有 status ∈ {parsed, failed} 的教材批量触发抽取。
**响应**:`{"started": ["03_生理学", "..."]}`

### `GET /api/graph/{book_id}`
- `book_id` = 单本教材 id → 单本图谱
- `book_id = "master"` → 整合后的主图谱
**响应**:`BookGraph`(完整 schema 见 系统设计.md §4)
```json
{
  "book_id": "03_生理学",
  "book_title": "生理学",
  "nodes": [{"id":"...", "name":"动作电位", "definition":"...", ...}],
  "edges": [{"source":"...", "target":"...", "relation_type":"prerequisite", ...}]
}
```

## 4. 跨教材整合

### `POST /api/integration/run`
**响应**:`{"status": "started"}`(异步,内部锁防止并发)
**409**:已有任务运行中

### `GET /api/integration/status`
**响应**:`{"running": false}`

### `GET /api/integration/decisions`
**响应**
```json

  "decisions": [
    {
      "decision_id": "d_a3f2",
      "action": "merge",
      "affected_nodes": ["...", "..."],

```

## 5. RAG 问答

### `POST /api/rag/index`
全量重建向量索引(对所有 parsed 教材)。
**响应**:`{"status": "started"}`(异步)

### `GET /api/rag/status`
**响应**
```json
{
  "indexed_books": 7,
  "total_chunks": 5230,
  "books": ["01_局部解剖学", "..."],
  "indexing": false,
  "progress": ""
}
```

### `POST /api/rag/query`
**请求**
```json
{"question": "动作电位的产生机制是什么?", "top_k": 5}
```
**响应**:`QAResponse`
```json
{
  "answer": "动作电位是细胞受刺激后...",
  "citations": [
    {"textbook":"生理学","chapter":"第二章 细胞的基本功能","page":35,"relevance_score":0.92},
    {"textbook":"病理生理学","chapter":"第三章","page":78,"relevance_score":0.83}
  ],
  "source_chunks": ["细胞处于静息状态时...", "..."]
}
```
**400**:question 为空

## 6. 多轮对话

### `POST /api/chat`
**请求**
```json
{"message": "为什么把炎症和炎症反应合并了?", "session_id": "abc123"}
```
session_id 可选;不传时自动生成新会话。

**响应**:`ChatResponse`
```json
{
  "session_id": "abc123",
  "reply": "因为《生理学》和《病理学》的'炎症'描述高度相似...\n\n✅ 已更新 1 项决策(图谱已实时刷新)。",
  "decisions_changed": ["d_a3f2"],
  "history": [
    {"role":"user","content":"...","timestamp":1715392845.123},
    {"role":"assistant","content":"...","timestamp":1715392847.456}
  ]
}
```

`decisions_changed` 非空表示主图谱已被修改,前端应重新拉 `/api/graph/master`。

---

## 7. 错误响应格式

所有错误统一返回 FastAPI 标准格式:
```json
{"detail": "错误信息"}
```
状态码:400(参数)/ 404(未找到)/ 409(冲突)/ 500(内部)

---

## 8. 调用示例(curl)

```bash
# 上传
curl -X POST http://localhost:8000/api/upload \
  -F "files=@03_生理学.pdf" -F "files=@05_病理学.pdf"

# 抽取所有
curl -X POST http://localhost:8000/api/graph/extract_all

# 整合
curl -X POST http://localhost:8000/api/integration/run

# 索引
curl -X POST http://localhost:8000/api/rag/index

# 提问
curl -X POST http://localhost:8000/api/rag/query \
  -H 'Content-Type: application/json' \
  -d '{"question":"动作电位是什么?"}'

# 对话
curl -X POST http://localhost:8000/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"message":"保留免疫应答"}'
```
