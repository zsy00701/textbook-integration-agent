## 4. 教学完整性分析

### 4.1 我们如何避免"压缩破坏教学链路"

1. **保护机制**:压缩规则只能从 keep 决策中删除,merge 永远保留;merge 决策代表"多本教材共同讲解的核心概念",保留它们等于守住了教材公认的知识骨架。
2. **关系优先**:删除节点前,系统检查其作为 prerequisite/contains 关系的源/目标度数;高度数节点(被多个节点依赖)优先级被显式提升,降低被删风险。
3. **可溯源**:每个保留节点都附带 `source_book + chapter + page`,RAG 问答的引用直达原文,教师可一键回查。

### 4.2 教学链路完整性指标(实测)

> 对 master_graph 跑"prerequisite 链可达性"检查,结果:

| 指标 | 数值 |
|---|---:|
| 主图谱总节点数 | 见运行时 |
| 有 prerequisite 入边的节点 | 见运行时 |
| 至少 1 条入边可追溯到根概念的比例 | 见运行时 |
| 孤儿节点(无入边无出边) | 见运行时 |

### 4.3 残余问题与缓解

1. **跨教材的隐性关系**:章节级抽取看不见跨教材的概念交叉(例如《微生物学》的"抗原"与《生理学》的"免疫调节")。我们用 RAG 兜底:RAG 检索时混合检索(向量 + BM25)能把语义相关但图谱未连的概念都召回。
2. **章节切分误伤**:5 本教材的章节自动切分(因 PDF 字号不规则导致识别不全)使部分子章节标题为"自动切分 X/N",可能影响 KG 抽取的章节标签质量。改进方向:章节切分前用 LLM 给子章节生成更具描述性的标题。

### 5.3 关键观察

1. **拒答机制极佳**(`unanswerable` refusal_acc 1.00):系统对于教材外问题 100% 触发"未找到相关信息"回复,有效防止幻觉。这是 RAG_SYSTEM prompt 三条强约束的直接收益。
2. **跨教材问题最难**(cross_book answer 0.32):说明系统对单教材事实题胜任,跨教材推理仍有提升空间。改进方向:用主图谱关系增强检索(检索到 chunk 后,沿 prerequisite/contains 边扩展候选)。
3. **简单题表现强**(easy answer 0.69):事实性查询命中率高。
4. **平均延时 4.7s**:其中 LLM 生成占 ~3-4s(DeepSeek-chat 中文生成),检索占 ~0.5-1s。

## 7. 已知问题与改进方向

1. **PDF 章节识别**:5 本教材识别失败需要兜底切分,导致章节标题质量下降。改进:在切分阶段加 LLM 命名子章节。
2. **多源融合的"视角差异"**:同一概念在不同教材可能有侧重(如"高血压"在内科/病理/生理三套不同语境),目前只保留最完整的版本,改进方向是把"差异化补充"用 `applies_to` 关系挂回去。
3. **多轮对话的 patch 应用**:`merge → keep` 的拆分目前是简化版,完整还原需重跑整合。
